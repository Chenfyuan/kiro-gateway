# kiro-gateway 管理界面功能规划

## 一、仪表盘（Dashboard）
- 实时请求数（今日/本周/本月）
- Token 用量统计（input/output/total，按天折线图）
- 费用估算（按模型定价计算）
- 当前活跃账号数 & 健康状态
- 最近请求列表（实时滚动）

## 二、账号管理
- 账号列表（状态、区域、请求数、失败率）
- 添加/删除账号
- 账号健康监控（连续失败次数、最后成功时间）
- 手动刷新 token / 重置失败计数
- 账号优先级 & 负载均衡权重配置

## 三、用量统计 & 费用
- 按时间维度：小时/天/周/月
- 按模型维度：opus-4.6 / sonnet-4 / 等
- 按 API Key 维度（多租户场景）
- Token 明细：input_tokens、output_tokens、cache_read
- 费用估算（可配置各模型单价）
- 导出 CSV/Excel

## 四、API Key 管理
- 多 Key 支持（给不同用户/应用分配不同 Key）
- 每个 Key 的用量配额（日/月限额）
- Key 用量独立统计
- Key 启用/禁用

## 五、请求日志
- 请求/响应记录（可选开启）
- 按时间、模型、状态筛选
- 慢请求标记（响应时间 > 阈值）
- 错误请求详情（4xx/5xx）
- 日志保留策略（自动清理）

## 六、配置管理
- 环境变量在线编辑（无需重启）
- 功能开关（FAKE_REASONING、TRUNCATION_RECOVERY 等）
- Model alias 配置
- 调试模式开关

## 七、告警 & 通知
- 账号全部失败告警
- 请求错误率超阈值告警
- Token 用量超预算告警
- 通知渠道：邮件 / 微信 / Webhook

---

## 技术方案

| 层 | 方案 |
|---|------|
| 前端 | React + Tailwind（或 Vue），轻量 SPA |
| 后端 | 复用现有 FastAPI，新增 `/admin/` 路由 |
| 存储 | SQLite（单机够用）或 PostgreSQL（多实例） |
| 认证 | 独立管理员密码，与 API Key 分离 |

## 优先级

- **P0（先做）：** Token 用量统计 + 费用估算 + 请求日志
- **P1（其次）：** API Key 多租户 + 配额管理
- **P2（再后）：** 仪表盘可视化 + 告警通知
- **P3（锦上添花）：** 在线配置管理 + 账号管理 UI 化
